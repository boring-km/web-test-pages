01.Executive Summary

핵심 발견 8가지

가장 큰 문제는 "내 서점 vs 히스토리"의 정보 구조 신호가 부서져 있고, 레트로 컨셉을 지탱해야 할 타이포·컬러·네이밍 규칙이 화면마다 다르게 적용된다는 점. 사용자가 "독서 시작" 이후 책이 사라졌다고 느끼는 현상은 개별 버그가 아니라 이 문제의 표면 증상입니다.

Top Takeaway Quick Win 10건만 먼저 반영해도 체감이 크게 바뀝니다. 영향도 🔴 4건은 네이밍·날짜 포맷·번호 체계 같은 코드 1~2줄 수정 수준. 위젯 개발은 CC-1(네이밍) · HOME-2(안정 식별자) 해결 후 착수를 권장합니다.

CC-4 영문 UPPER 헤더·버튼 남용 Concept ▾

계획서상 타겟은 전 연령대. 중장년층은 영문+letterSpacing 대문자에 인지 부하 급증. 80~90년대 한국 컴퓨터 경험은 한글 도스 프롬프트·하이텔·천리안 — 한글+픽셀도 동일 컨셉입니다.

개선안

  • START → 시작, FINISH → 끝, BOOK NAME → 책 이름
  • [+] ADD BOOK → [+] 책 추가 (하단 탭과 용어 통일)
  • 픽셀 폰트의 독특함은 한글에서도 살아나므로 한글 대체가 컨셉을 약화시키지 않음
CC-5 읽기 전용 박스 ↔ 입력 필드 어포던스 혼선 Med ▾

AddBookScreen(바코드·검색 결과)과 ManualBookInputScreen(수동 입력)이 같은 PixelShadowBox + 흰 배경 스타일을 공유. 전자는 Text만, 후자는 BasicTextField. 시각적으로 구분 불가.

개선안

  • 편집 가능: 그림자 박스 + 우측 연필 아이콘, 포커스 시 테두리 강조
  • 읽기 전용: 그림자 박스 없이 BackgroundGray, 라벨만 강조
CC-6 타이포 혼용 규칙 부재 Med ▾

현재 규칙 추정: 고정 라벨·타이틀 = 픽셀, 사용자 데이터·본문 = 고딕. 합리적이지만 BookInfo의 SAVE/START/FINISH는 라벨인데 영문 대문자, 약관만 유일하게 WantedSansBodySmall 등 예외가 많음. DungGeunMoBody.copy(letterSpacing = 3.2.sp) 같은 즉석 변형이 화면마다 흩어져 있음.

개선안

  • 테마 내 스타일 토큰에 "언제 쓰는가" 한 줄 주석
  • DungGeunMo 7단계 → 5단계로 축소: HomeTitle / Header / Subtitle / Body / Tag
CC-7 접근성 공통 이슈 Med ▾
  • 터치 영역: 설정 버튼 30dp, 드롭다운 화살표 14dp. WCAG 권고 48×48dp 미달
  • contentDescription = null: HomeScreen.kt:144/206, SearchBookScreen.kt:228 등. 픽셀 아이콘들 스크린리더 미지원
  • 색 대비: TextPrimary.copy(alpha = 0.5f) 등 낮은 알파 텍스트 WCAG AA 4.5:1 검증 필요
  • 문자 X 닫기 버튼 (MainScreen.kt:116): contentDescription 없음

문제

사용자가 "독서 시작"을 누르면 책이 서점에서 사라지고 "히스토리"에 편입. 사용자 직접 보고: "책이 꼭 사라진 것만 같음. 히스토리가 다 읽은 게 아니라 읽는 중인데도 완독처럼 보임."

  4. 탭 아이콘에 읽는 중 개수 배지
