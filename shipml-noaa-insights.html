CHAPTER A · congestion

항만은 왜 막혀 있나

각 배가 항만에 도착해 부두에 묶이기 전, 도착 직전 24시간의 AIS 신호를 봅니다. SOG가 1노트 미만으로 머문 시간을 모두 더하면 그게 "기다린 시간"입니다. 사실상 바다 위에서 떠 있던 시간.

16.4시간
상위 10개 항만에 도착하는 배가 부두에 묶이기 전 평균적으로 기다리는 시간. 중앙값으로는 20시간. 도착의 27%는 6시간 이상, 14%는 12시간 이상을 기다렸다.

방법론

# 도착 직전 24시간 동안 SOG < 1.0 노트인 시간을 합산
sub = df[(df.MMSI == mmsi)
         & (df.BaseDateTime >= arrival - 24h)
         & (df.BaseDateTime < arrival)]

deltas = sub.BaseDateTime.diff().dt.total_seconds() / 60
is_waiting = sub.SOG < 1.0
# 1시간 이상의 신호 gap은 제외 (AIS 침묵 영향 차단)
wait_min = deltas[is_waiting & (deltas < 60)].sum()

언제가 가장 막히나?

오른쪽은 도착 시각의 시간대 × 요일로 평균 대기 시간을 그린 heatmap. 색이 진할수록 더 오래 기다린다는 뜻.

주목할 패턴 두 가지:

  • 주말 도착이 평일보다 길게 기다림 — 부두 작업 인력이 적어서일 가능성
  • 새벽~이른 아침 도착은 의외로 대기가 짧음 — 부두가 비어 있어 바로 들어감

실용 시사점: 당신이 LA항에 도착할 화물을 보낸다면, 새벽 4~6시 도착이 가장 빠르게 부두 접안된다 — 적어도 이 15일치 데이터에선.

CHAPTER B · silence + ML

왜 어떤 배는 신호가 사라지나

AIS는 정상이라면 2초~몇 분에 한 번씩 신호를 보냅니다. 우리 데이터에서 30분 이상 신호가 끊긴 구간을 모두 모으니 9,570건의 "침묵"이 나왔습니다.

침묵은 세 가지 원인이 가능합니다:

  1. 음영 지역 — 산·건물·기상 영향으로 수신소가 못 받음
  2. 장치 끔 — 정박 중 절전 모드, 또는 의도적 종료
  3. 회피 행위 — 불법 어업, 제재 회피, 추적 차단

1단계: 룰 기반 탐지

2단계: 비지도 ML로 비정상 패턴 학습 (IsolationForest)

룰 기반은 한계가 있습니다. "6시간 이상" 같은 임계값은 자의적이고, 카테고리·시간대·위치를 다 같이 보지 못합니다. 이걸 9,570건의 침묵 이벤트를 통째로 IsolationForest에 넣어 자동으로 비정상 패턴을 학습하게 했습니다.

# 비지도 학습 — 라벨 없이 "흔치 않은 패턴"을 자동 탐지
model = IsolationForest(
    n_estimators=300,
    contamination=0.05,   # 5%를 비정상으로 가정
    random_state=42,
)

# 입력: 침묵 길이, 위치 점프, 추정 속도, 시각, 선종 (one-hot)
features = ["log_silence_h", "log_jump_km",
            "log_speed_kn", "hour", "VesselCategory"]
model.fit(X[features])

# 결과: anomaly_score (낮을수록 비정상)
silences["is_anomaly"] = model.predict(X) == -1
# → 9,570건 중 479건(5.0%)이 비정상으로 자동 분류

왜 IsolationForest인가

일반적인 분류 모델(RandomForest 등)은 "정상/비정상" 라벨이 필요합니다. 하지만 우리는 어떤 침묵이 비정상인지 알려주는 라벨이 없습니다. 그래서 unsupervised learning을 씁니다.

IsolationForest는 "이 점이 다른 점들로부터 얼마나 빨리 격리되는가"를 트리로 측정합니다. 구석에 외롭게 떨어진 점일수록 빨리 격리되고, 그게 곧 이상치라는 발상.

한계: 모델이 "비정상"이라고 한 게 진짜 회피 행위인지, 아니면 단순한 음영 지역인지는 우리가 검증할 수 없습니다. 이 결과는 "우선 들여다볼 후보 명단"으로 쓰는 게 맞습니다.

CHAPTER C · network

항만끼리는 어떻게 연결돼 있나

15일 동안 발생한 정박 이벤트 7,693건을 시간순으로 배열해, (현재 항만 → 다음 항만) 페어 1,057개를 만들었습니다. 이걸 그래프로 보면 60개 항만이 192개 고유 경로로 연결된 네트워크가 됩니다.

이 네트워크는 우리의 다음-항만 예측 모델이 학습하는 정답지 그 자체입니다. 모델이 78%의 정확도를 내는 이유가 바로 이 그래프 안에 있는 강한 패턴(자주 다니는 경로) 때문.

SYNTHESIS

세 인사이트가 합쳐서 말해주는 것

각각의 인사이트는 그 자체로도 의미 있지만, 셋을 합쳐 보면 LA·롱비치 항만 시스템의 운영 구조가 보입니다.

  1. A(혼잡도)는 LA항이 만성 적체 상태임을 보여주고,
  2. C(네트워크)는 그 적체가 #1↔#2를 비롯한 소수 핵심 항만에 집중돼 있음을 보여주며,
  3. B(침묵 + ML 이상 탐지)는 그 모든 운영의 사각지대 — 데이터가 안 들어오는 시간·장소 — 를 정량화한다.

이 세 가지를 같이 보지 않으면 우리 ML 모델 결과 해석이 어렵습니다:

  • 다음 항만 예측 78%가 인상적인 이유 → C가 보여주듯 트래픽이 소수 경로에 몰려 있어 학습 신호가 강함
  • 1시간 좌표 예측 1.41km가 가능한 이유 → A가 보여주듯 많은 시간 배가 한자리에서 떠 있음 (예측이 쉬움)
  • 선종 분류 76%의 천장 → B가 보여주듯 일부 카테고리는 침묵이 많아 학습 데이터 자체가 노이지함

이 보고서의 한계

  • 15일은 계절성·연중 변화를 못 본다. 항만 혼잡은 명절(추수감사절·크리스마스 직후) 직전이 피크라는 게 알려져 있는데, 1월 1~15일은 그 직후 회복기라 평소보다 차분할 수 있음.
  • 침묵의 진짜 원인은 검증 불가. ML이 "비정상"이라 한 479건이 실제 회피 행위인지, 단순히 산악 지형의 음영 지역인지는 외부 데이터(USCG 사고 기록·해안 지형) 없이는 모름.
  • 항만 이름은 추정. DBSCAN이 자동으로 만든 클러스터 ID에 실제 항만 명칭(예: "Pier J", "Pier T")을 매핑하려면 LA·LB 항만청 데이터셋과의 spatial join이 필요.
  • 이상 탐지의 contamination=0.05는 자의적 가정. "데이터의 5%는 비정상"이라는 사전 믿음을 모델에 주입한 것. 실제 비율이 1%면 위양성이 많고, 10%면 위음성이 많음.
