항만港灣 · port
배가 들어와 멈추고 화물·승객을 싣고 내리는 바닷가 시설. 부두·창고·크레인이 모여 있는 곳. 한국으로 치면 부산항·인천항. 이 보고서가 다루는 LA·롱비치는 미국에서 가장 큰 컨테이너 항만 두 곳이 붙어 있는 단지.
정박碇泊 · mooring / anchoring
배가 닻(anchor)을 내리거나 부두에 묶여서 한 자리에 멈춰 있는 상태. 항만 안에서 화물을 내리거나, 항만 입구의 앵커리지(정박지)에서 부두 차례를 기다릴 때 한다.
부두 접안berth / docking
배가 부두 옆 자리에 직접 묶이는 것. 컨테이너 하역은 부두 접안 후 가능. 부두 자리가 없으면 앵커리지에서 대기 → 그게 이 보고서 A 챕터의 주제.
knot노트 (속도 단위)
해상에서 쓰는 속도 단위. 1노트 ≈ 1.852 km/h. 정박 중인 배는 0~0.5노트, 항해 중인 화물선은 12~20노트, 빠른 페리는 30노트 이상.
데이터 컬럼 (AIS 신호 안의 정보)
AISAutomatic Identification System
국제 협약으로 모든 큰 배가 의무적으로 켜야 하는 위치 송신기. 30초~몇 분에 한 번씩 "나는 누구이고, 어디에 있고, 얼마나 빠르게 어느 방향으로 가고 있다"를 무선으로 송신. 자동차의 GPS는 받기만 하지만 AIS는 외부에 쏨.
MMSIMaritime Mobile Service Identity
배의 주민등록번호. 9자리 숫자. 같은 MMSI = 같은 배. 우리가 "1,859척"이라고 말하는 건 고유 MMSI 1,859개라는 뜻.
SOGSpeed Over Ground
실제 지면(해면) 기준 속도. 노트(knots) 단위. 우리가 "정박" 또는 "기다림"을 정의할 때 SOG < 1노트 같은 임계값을 씀.
VesselType선종 코드
배 종류를 나타내는 숫자 코드. 70번대=화물선(Cargo), 80번대=유조선(Tanker), 60번대=여객선(Passenger), 30번=어선(Fishing). 국제전기통신연합(ITU)이 정한 표준.
분석 방법론 / 머신러닝
BBOXbounding box / 경계 상자
지도 위에 사각형을 그어 그 안의 점만 남기는 작업. 미국 전체가 아니라 LA·롱비치만 보기 위해 위도 33–34° / 경도 -119 ~ -117.5°라는 박스를 사용.

지구가 둥근 걸 고려해서 두 위경도 점 사이의 실제 거리(km)를 계산하는 공식. 위경도 차이를 그냥 직선으로 계산하면 적도와 극지방에서 오차가 커짐.
DBSCANDensity-Based Spatial Clustering
점들이 빽빽이 모여 있는 곳을 자동으로 한 묶음으로 만드는 군집화 알고리즘. 사람이 "그룹을 몇 개로 만들지" 정해주지 않아도 됨. 우리는 이걸로 정박 좌표들을 묶어 "항만 76곳"을 자동 식별.
IsolationForest비지도 이상 탐지 ML
"정상/비정상" 라벨이 없는 데이터에서 "이상한 점"을 자동으로 찾아내는 머신러닝 모델. 다른 점들로부터 빨리 격리(isolation)되는 점일수록 이상치라는 발상. B 챕터에서 침묵 이벤트 2만 건 중 5%(1,024건)을 자동 분류.
contamination이상치 비율 가정값
IsolationForest에 미리 알려주는 "데이터의 몇 %가 비정상이라고 생각하느냐"라는 사전 믿음. 우리는 0.05(5%)로 설정. 이 값이 자의적이라는 점이 이 ML 결과의 한계.
ML 모델machine learning model

CHAPTER A · congestion

항만은 왜 막혀 있나

각 배가 항만에 도착해 부두에 묶이기 전, 도착 직전 24시간의 AIS 신호를 봅니다. SOG가 1노트 미만으로 머문 시간을 모두 더하면 그게 "기다린 시간". 사실상 바다 위에서 떠 있던 시간입니다.

16.3시간
상위 10개 항만에 도착하는 배가 부두에 묶이기 전 평균적으로 기다리는 시간. 중앙값으로는 20시간. 도착의 80%가 6시간 이상, 72%가 12시간 이상을 기다렸다. 31일치로 확장하니 단기(15일치) 분석에서 보였던 27%보다 훨씬 심각함을 확인.

방법론

# 도착 직전 24시간 동안 SOG < 1.0 노트인 시간을 합산
sub = df[(df.MMSI == mmsi)
         & (df.BaseDateTime >= arrival - 24h)
         & (df.BaseDateTime < arrival)]

deltas = sub.BaseDateTime.diff().dt.total_seconds() / 60
is_waiting = sub.SOG < 1.0
# 1시간 이상의 신호 gap은 제외 (AIS 침묵 영향 차단)
wait_min = deltas[is_waiting & (deltas < 60)].sum()

CHAPTER B · silence + ML

왜 어떤 배는 신호가 사라지나

AIS는 정상이라면 2초~몇 분에 한 번씩 신호를 보냅니다. 우리 데이터에서 30분 이상 신호가 끊긴 구간을 모두 모으니 20,471건의 "침묵"이 나왔습니다. 1,859척 중 76%(1,409척)가 한 번 이상 침묵을 경험했습니다.

침묵은 세 가지 원인이 가능합니다:

  1. 음영 지역 — 산·건물·기상 영향으로 수신소가 못 받음
  2. 장치 끔 — 정박 중 절전 모드, 또는 의도적 종료
  3. 회피 행위 — 불법 어업, 제재 회피, 추적 차단

1단계 — 룰 기반 탐지

2단계 — 비지도 ML로 비정상 패턴 학습 (IsolationForest)

룰 기반은 한계가 있습니다. "6시간 이상" 같은 임계값은 자의적이고, 카테고리·시간대·위치를 다 같이 보지 못합니다. 이걸 20,471건의 침묵 이벤트를 통째로 IsolationForest에 넣어 자동으로 비정상 패턴을 학습하게 했습니다.

# 비지도 학습 — 라벨 없이 "흔치 않은 패턴"을 자동 탐지
model = IsolationForest(
    n_estimators=300,
    contamination=0.05,   # 5%를 비정상으로 가정
    random_state=42,
)

# 입력: 침묵 길이, 위치 점프, 추정 속도, 시각, 선종 (one-hot)
features = ["log_silence_h", "log_jump_km",
            "log_speed_kn", "hour", "VesselCategory"]
model.fit(X[features])

# 결과: anomaly_score (낮을수록 비정상)
silences["is_anomaly"] = model.predict(X) == -1
# → 20,471건 중 1,024건(5.0%)이 비정상으로 자동 분류

왜 IsolationForest인가

일반적인 분류 모델(RandomForest 등)은 "정상/비정상" 라벨이 필요합니다. 하지만 우리는 어떤 침묵이 비정상인지 알려주는 라벨이 없습니다. 그래서 unsupervised learning을 씁니다.

IsolationForest는 "이 점이 다른 점들로부터 얼마나 빨리 격리되는가"를 트리로 측정합니다. 구석에 외롭게 떨어진 점일수록 빨리 격리되고, 그게 곧 이상치라는 발상.

한계: 모델이 "비정상"이라고 한 게 진짜 회피 행위인지, 아니면 단순한 음영 지역인지는 우리가 검증할 수 없습니다. 이 결과는 "우선 들여다볼 후보 명단"으로 쓰는 게 맞습니다.

세 인사이트가 합쳐서 말해주는 것

각각의 인사이트는 그 자체로도 의미 있지만, 셋을 합쳐 보면 LA·롱비치 항만 시스템의 운영 구조가 보입니다.

  1. A(혼잡도)는 LA항이 만성 적체(80%가 6h+ 대기)임을 보여주고,
  2. C(네트워크)는 그 적체가 #1(롱비치 메인)을 비롯한 소수 핵심 항만에 집중돼 있음을 보여주며 — 동시에 #22↔#20 같은 페리 노선처럼 일부 경로는 별도 운영되고 있음을 드러낸다,
  3. B(침묵 + ML 이상 탐지)는 그 모든 운영의 사각지대 — 데이터가 안 들어오는 시간·장소·선종 — 를 정량화한다. 특히 유조선·어선·여객선의 비정상 침묵 비율이 높다는 점은 후속 조사 가치가 있다.

이 세 가지를 같이 보지 않으면 우리 ML 모델 결과 해석이 어렵습니다:

  • 다음 항만 예측 78%가 인상적인 이유 → C가 보여주듯 트래픽이 소수 경로에 몰려 있어 학습 신호가 강함 (top route는 같은 7척이 121번 반복)
  • 1시간 좌표 예측 1.41km가 가능한 이유 → A가 보여주듯 많은 시간 배가 한자리에서 떠 있음 (예측이 쉬움)
  • 선종 분류 76%의 천장 → B가 보여주듯 일부 카테고리(특히 어선·유조선)는 침묵이 많아 학습 데이터 자체가 노이지함

이 보고서의 한계

  • 한 달치는 여전히 계절성을 못 본다. 항만 혼잡은 명절(추수감사절·크리스마스 직후) 직전이 피크라는 게 알려져 있는데, 1월 한 달은 그 직후 회복기일 가능성. 진짜 적체는 더 심각할 수 있음.
  • 침묵의 진짜 원인은 검증 불가. ML이 "비정상"이라 한 1,024건이 실제 회피 행위인지, 단순히 산악 지형의 음영 지역인지는 외부 데이터(USCG 사고 기록·해안 지형) 없이는 모름. 이 결과는 후속 조사 후보 명단으로 쓰는 게 맞음.
  • 항만 이름은 추정. DBSCAN이 자동으로 만든 클러스터 ID에 실제 항만 명칭(예: "Pier J", "Pier T")을 매핑하려면 LA·LB 항만청 데이터셋과의 spatial join이 필요.
  • 이상 탐지의 contamination=0.05는 자의적 가정. "데이터의 5%는 비정상"이라는 사전 믿음을 모델에 주입한 것. 실제 비율이 1%면 위양성이 많고, 10%면 위음성이 많음.

다음 작업으로 가능한 것

한 달치를 다 분석한 지금 시점에서 남은 큰 가지는: (1) ML 이상 탐지 결과를 USCG 사고 기록과 spatial join해 검증, (2) 항만 ID → 실제 명칭 매핑(LA Port Authority 데이터), (3) 1년치로 확장해 계절성·추수감사절 효과 측정, (4) Tanker↔Cargo 혼동을 줄이기 위한 시퀀스 모델(LSTM/Transformer) 비교.
